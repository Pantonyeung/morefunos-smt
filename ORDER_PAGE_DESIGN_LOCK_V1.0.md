# MoreFun SMT 點單頁設計鎖 V1.0

日期：2026-07-20  
狀態：點單頁第一層設計確認；程式實作與實機驗收待下一階段  
適用：橫屏 SMT POS、右手操作

## 1. 鎖定邊界

本文件鎖定點單頁主畫面、第一層卡片、抽屜、彈出提示及其互動規則。卡內第二層按鈕所開啟的詳細流程（例如設備維修、完整日結）不屬於本輪。

效果圖只代表設計與狀態，不代表功能已實作。正式完成必須通過「點擊、內容顯示、選擇、確認、主畫面更新、重新載入後保存」完整垂直鏈路。

## 2. 全局畫面與人體工學

- 以右手操作為基準；主要操作固定右下，取消或放棄放左下。
- 除快捷飲品細卡外，展開卡標準寬度為全畫面 25%。
- 卡片高度隨內容調整，最高不超過購物車可視高度；超出內容只在中間區域滾動，標題與底部操作固定。
- 卡片貼近觸發按鈕或項目展開；連體三角指向實際觸發位置，靠近邊緣時可翻轉方向。
- 卡片開啟後背景產品與按鈕不可操作；點擊空白背景無效。
- `×` 等同取消或返回；只讀卡可等同關閉。
- 需要保存的卡：左下取消、右下確認／儲存。只有單一關閉行為：右下單一返回／確認，左側不留假按鈕格。

## 3. 點單主畫面

### 3.1 頂部狀態列

- 訂單號即流水號，不另顯示流水號。
- 顯示接單／營業狀態。
- `待處理 n`：使用新訂單通知圖標與數量徽章，打開待處理卡。
- `線上`：使用 Wi-Fi／線上圖標，表示磨飯 App 連結與接單狀態，打開線上接單卡。
- `售罄 n`：只讀售罄列表。
- `快捷 ON/OFF`：快捷點單設定。
- `設備 n`：只讀設備狀態。
- `•••`：本機畫面設定，不等同底部「更多」。

### 3.2 購物車

- 購物車／產品區比例可選 25/75、30/70、32/68。
- 價格必須清楚且放大；數量控制放價格下方，「修改」緊鄰數量控制。
- 產品名稱下顯示設定摘要、描述或套餐內容；產品 Code 可由後台隱藏。
- 產品按分類即時分組，例如飯糰、飯餐、套餐；分類標題顯示「款數／件數」。
- 分類次序可由後台設定；分類內預設保留加入次序。
- 購物車可設定「合併展示／獨立展示」。只有產品、設定、備註、價格、套餐歸屬與包裝要求完全相同才可合併。
- 合併項目只修改其中一份時，使用「拆分修改 1份」。
- 套餐 Link Up 後移入「套餐」分類，原分類不重複顯示組件。
- 底部待整理狀態及結帳按鈕固定。

### 3.3 產品卡模式

提供三種裝置／使用者顯示偏好：

1. 純文字細卡：名稱、大價格、必要狀態，適合熟手快速點單。
2. 小圖細卡：小圖片、名稱、價格、一行副標題。
3. 大圖大卡：大圖片、名稱、價格、最多兩行描述／套餐內容。

三種模式不改產品次序；價格、售罄、必選及套餐標記必須保留。切換後保留目前分類及滾動位置。

## 4. 普通模式、快捷模式與訂單整理

### 4.1 普通模式

產品有必要設定時，先開產品設定卡，完成後加入購物車。

### 4.2 快捷模式

所有產品點擊後立即加入購物車，不因必選項目或套餐選項中斷。系統在背景識別可組合套餐、飯底、醬料、套餐飲品及其他必選項目。

購物車底部顯示動態狀態，例如：

`待整理 5項｜可組合 2｜必選 2｜飲品 1`

右側顯示「開始整理」。尚未完成時結帳按鈕改為「先整理 5項」，所有必選完成後才恢復正常結帳。

### 4.3 訂單整理次序

1. 套餐組合。
2. 飯底、醬料等產品必選。
3. 套餐飲品。
4. 飲品甜度、冰量與其他調整。

員工亦可由購物車個別項目的「修改」優先處理。

## 5. 產品設定卡

「必選卡、產品詳情、修改卡」合併為同一產品設定卡，只以新增、整理、修改三種入口狀態呈現。

內容包括：

- 產品名稱、價格、簡介、套餐內容與可展開的過敏原資料。
- 數量 `－／＋`；兩份完全相同直接設定 ×2。
- 必選項目：例如飯底、指定醬料。
- 其他調整：預設收起，點擊後才顯示走青瓜、走肉鬆、多醬、少醬、醬汁另上等。
- 備註：只用於預設選項無法表示的要求。

不同設定採「設定組合」展示；相同設定自動合併數量。產品已在購物車時，卡片標題必須顯示正在修改的產品名稱。

### 5.1 後台選項權重

後台可按全店、分類、指定產品設定：顯示次序／權重、必選或可選、單選或多選、最少／最多數量、預設值、快捷模式是否待整理、是否收納於其他調整、差價及暫停顯示。

套用優先次序：指定產品 > 產品分類 > 全店共用。

## 6. 套餐組合卡

系統偵測到購物車產品可 Link Up 時顯示：

`可組合 2套｜可節省 $16　[自由搭配] [一鍵組合 2套]`

「一鍵組合」為右側主要操作；「自由搭配」為左側次要操作。套餐組合卡由購物車提示向右展開，覆蓋部分產品區但不覆蓋購物車，寬度 25%，三角指回提示。

### 6.1 一鍵組合

系統提出套餐 A、B 建議，顯示內容、組合價及節省金額；員工確認後才改變價格與群組。

### 6.2 自由搭配

- 使用未分配產品 Pool。
- 先選 A，再選屬於 A 的主餐、飯糰、小食與飲品；完成後處理 B。
- 使用 A/B/C 字母及顏色雙重識別，不只依賴顏色。
- 每份產品數量只可分配一次；相同產品需要配不同套餐時自動拆分數量。
- 完成後購物車以套餐群組卡顯示，保留「修改配搭」。

## 7. 快捷飲品抽屜與飲品設定卡

### 7.1 快捷飲品抽屜

平時收起，只顯示 `快捷飲品｜待補 n⌃` 把手。有待補飲品時自動展開一次；由產品區底部向上覆蓋，最大約佔產品區 35%，不推高產品卡、不覆蓋購物車及底部導覽。

標題必須顯示歸屬，例如 `為套餐 B｜原味飯糰選飲品`；所選飲品只加入指定套餐。

### 7.2 飲品設定卡

- 寬度約為大產品卡的三分之二，連體三角指向觸發飲品卡。
- 平時只顯示已建立設定，例如走甜 ×1、少甜・少冰 ×1、少冰 ×1、無調整 ×1。
- 甜度與冰量選項只在「新增設定」或修改某行時展開。
- 不顯示正常甜／正常冰；沒有選擇代表「無調整」。
- 點一次選取，再點同一選項取消。
- 相同設定合併數量，不同設定分行；數量降至 0 移除該設定。
- 無任何選項的飲品只顯示名稱、數量與加入購物車。

## 8. 待處理、新訂單與線上接單

### 8.1 待處理卡

分為「新單」與「電話／WhatsApp 核對單」。中間分界自由上下移動：內容少的區域收縮、內容多的區域擴大；總高度不超過購物車，超出內容在各區內獨立滾動。

每張訂單顯示渠道 Logo／Link 來源、訂單號或電話尾號、件數、總金額、等候時間／狀態，以及處理／核對按鈕。操作由每行即時執行，底部不設套用，右下只設返回。

### 8.2 新訂單快捷通知

只在右下角顯示完整卡約 3 秒；提供「稍後處理」與「立即處理」。沒有操作時自動加入待處理列表，不會遺失或套入目前購物車。手指按住或正在點擊時暫停倒數；多張新單排隊並顯示餘下數量。

頂部待處理與新訂單只作 Link 來源、數量與入口顯示；未點「立即處理」前不載入購物車、不接單、不核對、不改狀態。

### 8.3 線上接單卡

卡片形式與快捷設定卡一致，標題「磨飯 App｜線上接單」。顯示連結狀態、同步狀態、最後同步時間及今日已接單量。

可設定：接單中、暫停接單、提早收工。提早收工可選立即、15/30/60 分鐘後或自訂停止接單時間。只有按「儲存設定」才同步到磨飯 App；取消或 `×` 不生效。

頂部結果可顯示：線上｜接單中、線上｜已暫停、線上｜18:30收單、離線｜待同步。

## 9. 其他第一層卡片

### 9.1 售罄卡

純只讀即時狀態列表；只顯示目前售罄產品及必要狀態，不提供查看、修改或恢復供應。供應狀態在後台「售罄設定」處理。右下單一確認。

### 9.2 快捷設定卡

設定普通／快捷模式、全部套餐飲品／自訂飲品、快捷飲品顯示及飲品排列。開啟後持續生效，必須手動關閉。排列使用上移／下移；修改後右下「儲存並開啟」。

### 9.3 設備狀態卡

第一層只讀顯示打印機、收款設備、網絡及同步狀態；詳細處理留待第二輪。右下單一確認。

### 9.4 畫面設定卡

右上 `•••` 只管理本機畫面：購物車比例、快捷飲品列顯示及尺寸、三種產品卡模式、圖片／副標題／Code、畫面密度、購物車合併／獨立展示與恢復預設。調整時即時預覽；取消還原、確認保存。

### 9.5 底部更多

底部「更多」是完整功能入口，包括訂單記錄、暫存單、交更／日結、收款記錄、售罄設定、產品及套餐、設備管理、使用者及權限；不與右上畫面設定重複。

## 10. 驗收門檻

每個入口必須驗證：觸發位置正確、卡片與三角指向正確、背景鎖定、內容可滾動、按鈕固定、取消不保存、確認保存、主畫面即時更新、重新載入後狀態仍正確。效果圖、按壓變色或靜態 HTML 均不可代替功能驗收。
