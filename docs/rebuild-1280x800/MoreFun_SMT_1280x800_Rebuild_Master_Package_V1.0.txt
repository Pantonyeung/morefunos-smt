0｜文件身份及權威聲明
======================================================================

本文件將 Source A（完整 SMT 功能／頁面／流程／商業邏輯）與 Source B（rebuild39、1280×800 點單頁、實機問題及重構限制）整合成唯一重建規格。

來源標記：
[A] 只來自完整 SMT 來源
[B] 只來自 1280×800 來源
[A+B] 兩邊一致
[A>B] A 較完整／功能權威
[B>A] B 較新／1280×800 視覺與工程權威
[衝突] 尚無足夠證據判定
[缺口] 兩邊資料不足
[否決] 使用者已否決
[錯誤實作] 已證明不可沿用

最高合併原則：
A 功能 + B 已確認 1280×800 表達 - B 補丁式工程 = 新 SMT 正式規格。

權威次序：
1. 使用者最後明確確認／修正／鎖定。
2. Source B 對 1280×800、rebuild39 實機及點單頁的確認。
3. Source A 最新獨立頁面 Lock。
4. Source A Master UI／Logic Lock／Slide Master Lock。
5. 舊交接、舊 Decision、舊程式，只作補充證據。

禁止：
- 直接拼接 A、B。
- 以 1920×1080 畫布縮放成 1280×800。
- 將 rebuild39 補丁底層直接當新架構。
- 因版本號較新而取代 rebuild39。
- 自行重新設計已鎖定商業邏輯。
- 未完成 Web／Sunmi 實機驗收先封裝 APK。
