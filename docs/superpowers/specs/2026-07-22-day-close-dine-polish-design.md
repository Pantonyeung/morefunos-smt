# 磨飯 SMT｜日結盤點、堂食取消及全局互動美化設計

狀態：產品負責人於 2026-07-22 對話確認；本輪實作依據。

## 目標

修正兩個會阻塞真實營運的流程，再把 SMT 各頁的觸控、按鈕、彈窗及轉場統一成簡潔、清楚、適合高峰操作的互動語言。

## 日結盤點

- 港幣面額固定包括紙幣 1000、500、100、50、20、10 元及硬幣 10、5、2、1、0.5、0.2、0.1 元。
- 每個面額同時提供數量及金額；修改任何一邊即時互推另一邊。金額必須是面額的完整倍數。
- 實際點算總額由全部面額相加，不再依賴單一手填總額。
- 保存開工底金、當日現金支出、收工提取及留底現金。提取加留底必須等於實際點算總額。
- 電話／WhatsApp、磨飯 App 等付款待核實訂單先歸入「待反推」，不可預先列作電子支付。
- 反推公式：
  - 已知櫃桶應有 = 開工底金 + 已核實現金銷售 − 現金支出。
  - 待核實現金 = 實點現金超出已知櫃桶應有的部分，上限為待核實總額。
  - 待核實非現金 = 待核實總額 − 待核實現金。
  - 差額 = 實點現金 −（已知櫃桶應有 + 待核實現金）。
- 可接受差額為當日淨銷售的 3%。超出時必須填寫原因及由目前終端明確授權通過；授權時間、終端及原因寫入不可覆寫日結版本與稽核。

## 堂食取消

- 空枱進入點餐時只建立點餐意圖，不提前建立使用中枱會話。
- 正式提交至少一項餐品後才開枱、建立批次及打印工作。
- 點單頁在堂食脈絡下清楚顯示枱號及「取消堂食點單」。有餐品時須二次確認；確認後清除購物車及堂食脈絡並返回堂食頁。
- 已使用枱的「加單」取消只取消今次加單，不清除原有枱資料。
- 堂食頁載入時清理舊版本遺留的空會話；結束枱時同步清除指向同一枱／會話的點單脈絡，避免殘留。
- 正式提交前仍會校驗會話編號；舊會話失效時拒絕掛錯枱。

## 全局互動美化

- 共用樣式統一最小觸控高度、圓角、邊框、陰影、焦點環及禁用狀態。
- 按鈕按下採輕微縮放及色彩回饋；主要按鈕保持橙色，危險操作保持紅色。
- 對話框淡入及輕微上移；右側抽屜由右滑入；遮罩只阻隔背景，不以點空白關閉重要操作。
- 對 `prefers-reduced-motion` 提供關閉位移及縮放的降級模式。
- 動效只用在狀態轉換及觸控確認，避免延長高峰操作時間。

## 驗收

- 純資料測試覆蓋面額雙向互推、總額、待核實反推、3% 覆核、提取／留底守恆及舊資料兼容。
- 堂食測試覆蓋空枱不提前開枱、正式落單才開枱、取消新開枱、取消使用中枱加單、舊空會話清理及失效會話拒絕。
- 介面測試確認日結欄位、授權操作、堂食取消控制及全局減少動效規則存在。
- 跑完整自動回歸；視覺以桌面及平板比例渲染，實體 Sunmi T2S 仍列為待實機驗收。
