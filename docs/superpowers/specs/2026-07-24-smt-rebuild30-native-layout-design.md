# SMT Rebuild30 原生 1280×800 還原設計

**範圍：** SMT 的來源盤點、版面真相還原與功能移植邊界；不包括正式 API、Firebase 規則、打印啟用、APK 發布，亦不改 Customer、Admin 或 SMM 的 UI。

## 已確認的版面決策

SMT 點單頁不是三欄 POS。

- 主工作區只有兩個主要操作區：左側購物車與右側產品目錄。
- 分類、搜尋及產品卡在產品目錄內。
- 快捷飲品不是獨立右欄；它是產品卡下方的快捷選擇飲品區。
- 收起時只保留產品目錄底部、導航上方的入口把手；展開時只在產品目錄內向上覆蓋，不得推動購物車、頂欄、底欄或結帳區。
- 待補仍以購物車內的 Required／Optional／Link Up 狀態與「整理」入口處理，不建立第三條永久欄位。

以上決策覆蓋整合主題包 §4.1 中「右側快捷飲品／待補」的文字描述；原因是產品負責人於 2026-07-24 明確澄清該描述不是實際版面。

## 來源與權威

1. 產品負責人 2026-07-24 最新確認：快捷飲品屬產品卡下方附屬區，不是第三欄。
2. `MoreFun_四端原生整合_新對話主題包_V1.0(1).txt`：只換骨、不換樣、原生 1280×800、禁止以一般三欄 POS 重設。
3. `pages/order/page.js` 及 `pages/order/page.css`（來源 `origin/product-design-rewrite-rebuild30`）：實際 DOM 已採 `cart + catalog` 兩區；`quickDrinks()` 位於 catalog 的產品卡之後。
4. `docs/design-lock-v1/ORDER_PAGE_CURRENT_LOCK.md` §2、§5、§8、§13：固定頂／底欄、購物車與產品區比例、快捷飲品收起／上拉、顯示設定。

## 技術設計

### 版面責任

| 區域 | 責任 | 不可改變 |
|---|---|---|
| 左側購物車 | 餐品、待補摘要、暫存／取單、結帳 | 25/75、30/70、32/68 比例設定；固定結帳區 |
| 右側產品目錄 | 分類、搜尋、產品卡、快捷飲品入口及展開面板 | 產品卡原有位置與操作；內容只在本區滾動 |
| 頂部／底部共用殼 | 狀態、設備、設定、五頁導航 | 永久固定，不被內容或抽屜遮蓋 |

### 功能移植原則

以 `83d95b0` 的頁面骨架作視覺基底；只逐項抽取 `feat/smt-order-page-v1`／`5d7f972` 已有的 Domain、資料合約及可驗證功能。每項移植必須同時符合以下條件：

1. 不增加第三欄或改變兩區 DOM 責任。
2. 不以 `transform: scale`、大量 `!important`、硬編碼高度或 z-index 壓住版面。
3. 先有失敗測試，再做最少實作，最後回歸全套測試。
4. 未有正式 API／硬件證據的能力只標示為本機鏈路或待驗收。

## 目前基線

- 工作分支：`rebuild30-native-1280-unified`
- 基底：`origin/product-design-rewrite-rebuild30`
- 基底提交：`83d95b030cc000271b0e8a5656385a31f1ca1a76`

既有失敗只屬後期功能與原版入口未對齊，不可用來改寫已確認版面：

- 堂食獨立路由
- 訂單獨立路由
- 更多頁與五項導航路由
- 售罄頁路由
- 固定 T2S 畫布與根高度／滾動鏈
- 子頁啟動錯誤可見後備頁

## 分段驗收次序

1. 先恢復固定 T2S 畫布、根高度與兩區內部滾動，保持現有視覺位置。
2. 接回各頁路由與子頁錯誤後備，不搬動點單區。
3. 逐項接回堂食、訂單、售罄、更多頁；各頁沿用共用頂欄與五項導航。
4. 最後才把已核對的點單 Domain、Firebase 餐牌讀取、暫存、付款、打印佇列等能力遷入；正式寫入、硬件打印與 APK 必須另行驗收。

- 不啟用正式流量、Firebase 規則、Worker secrets、正式打印或 production APK。
