# 購物車、動態配對與飲品視覺語言設計

## 已確認範圍

- 購物車每列維持單行視覺：圖片與內容在左，金額完全靠右；數量減／數量／數量加／修改向上收齊，不能因描述換行跌到下一列。
- 指定配對卡增加安全內距、邊框與字級。配對組數為 `min(可組合主餐數量, 小食數量)`，以 A、B、C…動態建立，每組獨立選主餐、小食及飲品。
- 快捷飲品抽屜使用直向高卡，名稱在圖片上方，卡高大於卡寬；選中以橙框、橙底與箭嘴指示。
- 產品修改、統一整理、快捷飲品抽屜共用相同飲品卡語言，只以尺寸變體適應使用位置。
- 產品修改內的飲品列常駐展開，不使用收合把手。

## 驗收

- Safari 實機不得再被後段 CSS 覆蓋成橫向矮卡。
- A–J 等動態配對組必須按可組合數量生成，不能固定只有一組。
- 所有飲品入口的名稱位置、選中顏色及箭嘴一致。
