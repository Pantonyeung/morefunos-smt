# 磨飯 SMT 點單頁｜現行設計鎖

狀態：已確認 V1.0（本輪實作唯一功能依據）  
目標分支：`feat/smt-order-page-v1`  
更新日期：2026-07-20

實作差距及驗收狀態見：[`ORDER_PAGE_IMPLEMENTATION_STATUS.md`](./ORDER_PAGE_IMPLEMENTATION_STATUS.md)。設計鎖只記錄「應該點做」，不得以程式存在或效果圖存在改寫本文件；實際完成程度只在狀態文件更新。

## 1. 文件權重

1. 舊 Project／SMT Lock 是最低基線，只約束安全、資料完整、離線可工作、繁體中文及基本可用性；新設計不得跌穿基線。
2. 本文件整理產品負責人在逐張卡審核中確認的尺寸、位置、內容、操作及新增功能，是本輪點單頁實作的唯一功能依據。
3. 本文件與舊基線衝突時，介面與流程以本文件為準；安全及資料完整不得低於舊基線。
4. 效果圖只作視覺證據；沒有寫入本文件的效果，不得視為已鎖定功能。
5. 「設計已確認」「程式已完成」「自動測試通過」「實機驗收通過」必須分開記錄。

## 2. 全局畫面規則

- 以 Sunmi T2S 工作比例為設計基準，iPad／其他橫屏裝置按可用安全區縮放。
- 頂部狀態欄及底部五項導航永久固定，不可被卡片、購物車或產品內容推動、遮蓋或移出畫面。
- 購物車、產品區及展開卡只可在自己的內容區內滾動。
- 內容增加時不得改變頂欄、底欄、結帳區及待補區位置。
- 支援手動縮放；縮放只改變整體顯示大小，不改變功能比例、定位規則或觸控邏輯。
- 點單工作區可設定購物車／產品區比例：25/75、30/70、32/68。
- 產品卡提供大圖大卡、小圖細卡、純文字細卡三種全局模式。
- 產品 Code 可隱藏；產品名稱下方可顯示產品描述或套餐內容；價格必須清楚及突出。

## 3. 全局展開卡規則

- 全畫面同一時間只容許一張主卡存在。
- 開啟新卡時，舊卡必須先關閉，禁止疊卡。
- 卡片開啟後，背景產品、分類、購物車及其他功能不可操作。
- 點空白位置無效，不得關閉卡片。
- `×` 等同返回；返回不保存，確認才保存。
- 有未套用變更時，返回／`×` 顯示放棄修改確認；沒有變更可直接返回。
- 主要確認按鈕固定在右下或底部右側；返回固定在其左側。
- 只有返回、沒有確認的狀態卡，返回按鈕放在右側，符合右手操作。
- 一般卡預設約佔工作區寬度 25%；內容多時增加高度，但不得高過購物車／安全工作區。
- 卡片內容過多時只有卡身內容滾動，標題及底部操作按鈕固定。

## 4. 卡片定位及箭嘴

- 卡片必須貼近觸發來源，箭嘴精確指向被按控制的中心。
- 頂欄來源：卡片在按鈕下方展開，箭嘴向上。
- 底部快捷飲品來源：卡片向上展開，箭嘴向下。
- 左側購物車來源：卡片向右展開，箭嘴向左。
- 右側產品／狀態來源：卡片向左展開，箭嘴向右。
- 接近邊界時可換方向，但箭嘴仍須指回實際觸發來源。
- 卡片不得越過頂部狀態欄或底部導航欄。

## 5. 購物車

- 訂單號就是流水號，不另外顯示流水號。
- 購物車按產品類別分組展示；分組次序由後台設定。
- 相同配置可合併或逐項分開展示，合併模式由設定控制。
- 每項顯示產品名稱、副標題／已選配置、總價。
- 購物車產品圖片可由員工顯示或隱藏；隱藏後不可留下圖片空位。
- 操作固定為同一組：`－　數量　＋　修改`。
- 加減、修改及價格不能因名稱或內容長度分離。
- 增加產品或選項時，只有購物車清單內部滾動；待補區、暫存、結帳及底部導航不動。
- 必選完成後主按鈕顯示 `結帳 $金額` 並前往付款頁，不使用「先處理」。

## 6. 產品點單模式

### 6.1 普通模式

- 按產品後開啟需要的必選／調整卡，完成確認後加入購物車。

### 6.2 快捷模式

- 按產品立即加入購物車，不因產品存在選項而阻塞。
- 未選的必選項進入待補流程；非必選調整日後可由購物車「修改」處理。
- 快捷模式開啟後保持啟用，直至員工手動關閉。

## 7. 修改產品卡

- 由購物車該項「修改」向右展開，箭嘴指向該行。
- 寬度約為工作區 25%，高度按內容調整。
- 標題必須顯示正在修改的產品名稱及目前價格。
- 顯示產品相關選項、備註及套餐內容。
- 必選、多選、可選但非必要的選項由後台設定；選項顯示次序／權重亦由後台設定。
- 不選代表標準；不長期展示「正常甜／正常冰」等沒有調整的選項。
- 底部固定「返回」「確認修改 $金額」。

## 8. 快捷飲品抽屜與飲品調整卡

- 快捷飲品平時收起，不長期佔用產品區；只顯示抽屜入口／把手。
- 按入口後向上展開快捷飲品抽屜，可覆蓋後方產品卡但不可越過導航。
- 快捷飲品卡寬度約為大產品卡的 2/3，高度按內容調整。
- 快捷飲品可選純文字或圖片模式；關閉圖片後不得留下圖片框。
- 飲品列可水平滑動，需清楚提示左右仍有內容。
- 飲品排列可由設定調整，支援上移／下移或數字排序；更改後即時反映抽屜次序。
- 快捷飲品列顯示、圖片模式、排列及快捷補選為互相獨立的設定。
- 按某飲品後，調整卡向上展開，箭嘴向下精確指向該飲品。
- 調整卡顯示飲品名稱、`－／數量／＋`；圖片是否顯示跟隨飲品圖片設定。
- 甜度及冰量預設收起／不選；不選即正常。
- 按選項一次選取，再按相同選項取消。
- 同款飲品不同配置可建立多個配置組，例如：正常×1、少甜少冰×2、走甜×1。
- 底部套用按鈕固定在卡內，不可超出卡片或底部導航。

## 9. 待補、必選及可組合套餐

### 9.1 三層規則

- Required（必選）未完成會阻止結帳。
- Optional（可補選／非必要調整）不阻止結帳。
- Link Up（可組合套餐）不阻止結帳。

### 9.2 統一整理入口

- 購物車底部顯示尚欠必選數量及可組合數量。
- 按「整理」開啟約25%卡，提供「必選補齊」及「可組合套餐」兩個清楚入口。
- 必選飲品可使用快捷飲品抽屜批量補齊。

### 9.3 可組合套餐卡

- 可將獨立飯糰、小食及飲品 Link Up 成套餐。
- 提供「自動組合」及「指定配對」兩種方式。
- 自動組合按 A、B、C 順序建立完整組合。
- 指定配對可指定某款小食／飲品屬於哪個飯糰或便當。
- 已配對數量與未配對餘量必須分開保留，不能把整行全部標記為已組合。
- 測試資料至少提供兩款可組合飯糰、兩款小食及三款飲品，讓實機立即驗證。

## 10. 待處理訂單卡

- 由頂部「待處理」向下展開，箭嘴向上指向按鈕。
- 沒有套用按鈕，只有右側返回／`×`。
- 分成兩個渠道區：
  1. 磨飯 App／網頁訂單。
  2. 電話／WhatsApp 排隊單。
- 每張訂單顯示：渠道圖標、訂單／排隊號、產品數量、金額、等候時間及聯絡資料（如有）。
- 訂單列可按入查看及處理。
- 兩區之間分界可動態移動：內容少的一區收細，另一區使用餘下空間。
- 整張卡總高度不可高過購物車；任何一區過多時在該區內部滾動。

### 10.1 訂單核對與狀態（2026-07-20 新標準）

- 訂單列先開摘要，按「開始核對」才進入完整核對卡；此時仍未正式接單。
- 核對卡顯示來源、訂單號、聯絡人、產品明細、產品數量、訂單金額、付款方式、付款狀態及付款證明。
- 付款證明可按下放大；未收到證明時不可按「確認接單」。
- 電話／WhatsApp 訂單提供 QR Code。公司電話掃描後，直接開啟該客人的 WhatsApp 對話及預設訊息；QR Code 不代表付款已確認。
- 核對完成按「確認接單」，訂單即成為後台「運行中訂單」，並由接受時間啟動30分鐘計時。
- 不設「製作中」或「待取餐」。正式狀態只使用：`待處理 → 運行中 → 已完成`。
- 運行中訂單滿30分鐘後由系統自動轉為已完成。

## 11. 售罄、設備與線上接單

### 11.1 售罄列表

- 頂部提供售罄狀態入口及數量。
- 只作即時預覽，顯示目前已售罄／停售產品。
- 不提供「恢復供應」；供應狀態在後台售罄管理設定。
- 只有返回／`×`。

### 11.2 設備狀態

- 由設備狀態按鈕附近展開，約25%寬。
- 顯示各設備／部件狀態及查看入口。
- 狀態確認與返回可合併為單一右側「完成／返回」。

### 11.3 線上接單卡

- 頂欄顯示線上圖標及接單狀態。
- 卡片型態與快捷設定卡一致，由線上按鈕下方向下展開。
- 支援磨飯 App／網頁接單開關、預定停止接單時間、提早收工、即時停止及恢復接單。

## 12. 快捷新訂單提示

- 新 App／網頁訂單提示只在右下角出現。
- 顯示約2–3秒；未操作則保留待處理角標及訂單。
- 提供「稍後處理」及「即時處理」。
- 快捷提示不是待處理完整列表，不能互相取代。

## 13. 顯示設定卡

- 頂部三點是點單頁視覺及快速操作設定，不等同底部「更多」。
- 包含：購物車比例、產品卡模式、產品 Code、購物車圖片、快捷飲品抽屜、飲品圖片、飲品排列、快捷補選及其他點單頁顯示開關。
- 底部「更多」保留低頻營運、設備、系統及後台功能。

## 14. 驗收閘門

每項功能必須分別記錄以下狀態，不可因效果圖或程式碼存在而標示完成：

1. 對話確認。
2. 寫入本設計鎖。
3. 實作完成。
4. 自動測試通過。
5. iPad Safari 實機通過。
6. Sunmi T2S 實機通過。
7. 產品負責人最終 Lock。

任何未完成第5至第7步的項目，只能標示「待驗收」，不得標示「完成／已鎖定」。

## 15. 購物車與飲品最終互動標準（order-v1-8）

- 購物車每列的價格使用獨立右側位置；數量減、數量、數量加與「修改」固定為同一操作組，不可隨描述長度漂移。
- 購物車圖片可開關；關閉圖片後名稱、描述、價格及操作自動填補空位，不保留假圖片欄。
- 相同產品展示可設定為「相同配置合併」或「逐項顯示」；配置不同不得誤合併。
- 快捷飲品收起時只顯示導航上方置中小型把手；展開時由下方向上打開直向飲品卡，選中卡必須以顏色及箭嘴標示來源。
- 飲品調整卡不重複產品圖片及長描述。預設為正常；只有按「新增調整」後才展開甜度、冰量及該組數量。
- 同款飲品可建立多組不同調整；再次按已選甜度或冰量即取消，未選代表正常。
- 指定配對使用每行三張純文字候選卡；飯團、小食、飲品分區選擇，選中狀態須清楚可見。

## 16. 實機對齊與共用飲品卡標準（order-v1-9）

- 購物車金額完全靠右；產品名稱放大。價格及加減／修改分別佔右側上下兩列，整組與圖片垂直齊平。
- 指定配對組數按 `min(可組合主餐數量, 小食數量)` 產生，最多 A–Z；每組獨立選擇主餐、小食及飲品，同一項不可超過購物車數量。
- 指定配對以 A、B、C…頁籤切換，避免大量組別令卡片無限增高；候選卡每行三張並保留安全內距。
- 快捷飲品、產品修改套餐飲品及統一整理飲品使用同一共用卡片：名稱置頂、圖片在下、選中時橙框及箭嘴。
- 快捷飲品使用最大直向卡；產品修改及統一整理使用同語言的小尺寸卡，並保持常駐展開。
